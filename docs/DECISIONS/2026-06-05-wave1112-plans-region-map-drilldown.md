# 2026-06-05 Wave 1112 - Plans Region Map Drilldown

## 결정

- 멤버십 신청 첫 화면의 지역 지도는 초기 상태에서 광역시/도 단위로만 읽히게 한다.
- 시/군/구 단위 경계와 대표 지역 텍스트는 사용자가 광역 지역을 선택한 뒤 확대 상태에서만 노출한다.
- 모바일 하단의 `수원시`, `성남시` 같은 지역 칩 목록은 제거한다. 좁은 화면에서는 지도와 남은 티오가 먼저 보여야 한다.

## 구현

- `PlansApplicationFlow`에 `mapZoomed` 상태를 추가해 전국 지도 상태와 선택 지역 상세 상태를 분리했다.
- `KoreaSeatMap`은 초기에는 내부 경계선을 지역 색으로 묻고, 선택된 광역 지역에만 상세 경계선을 다시 켠다.
- 지역을 클릭하면 해당 광역 지역 중심으로 지도를 확대하고, 선택 지역의 남은 티오를 크게 표시한다.
- 데스크톱 우측 패널도 초기에는 전체 남은 자리/예약률을 보여주고, 선택 후에는 선택 지역 기준으로 바뀐다.
- 티오가 찰수록 파랑/초록에서 주황/빨강으로 변하게 색상 체계를 바꾸고, 지도 상단에 여유→과밀 범례 바를 추가했다.
- 서울·인천·부산 등 면적이 작은 광역시는 선택 시 더 큰 줌 배율을 적용해 구 단위 경계가 보이도록 했다.
- 전국 단계의 광역 라벨 원과 글씨 크기를 키워 모바일에서도 지역명이 읽히게 했다.
- 지도 면, 경계선, 라벨을 별도 SVG 레이어로 분리해 경계선과 지역명 뱃지가 과밀 색상 면에 묻히지 않게 했다.
- 예약 진행바는 단색으로 단순화하고 다크모드 미충전 트랙을 더 밝게 조정했다.
- 티오 색상 범례는 상단 가로 바에서 지도 옆 작은 세로 범례로 이동했다.
- 상단 예약 진행바는 제거하고, 현재 예약 배지를 헤드라인과 같은 행에 정렬해 지도에 더 많은 세로 공간을 배정했다.
- 모바일 선택 후 별도 선택 지역 카드는 제거하고, 지도 안 큰 말풍선으로 선택 지역과 남은 티오를 강조한다.
- 지역 확대 후에는 hover 상태를 무시하고 선택한 지역의 남은 티오를 고정 표시한다.
- 서울 등 작은 광역시는 확대 배율을 더 높이고, 확대 후 구/시 단위 티오 선택 바를 지도 위에 표시한다.
- 세부 티오 선택 시 지도 말풍선과 우측 요약이 해당 구/시 기준으로 갱신된다.
- 전국 지도 색상은 과밀도 의미가 아니라 광역시/도 구분용 팔레트로 변경했다.
- 전국 지도 라벨을 다시 항상 노출하고 크게 키워 광역 단위 이름이 바로 보이게 했다.
- 전국 지도 뱃지는 지도 클릭 중심점과 별도의 표시 좌표를 두어 수도권/충청권/영남권 라벨이 서로 겹치지 않게 정렬했다.
- 광역 지역 선택 후 세부 구/시 티오는 하단 텍스트 칩 대신 지도 위 클릭 가능한 뱃지로 표시한다.
- 확대 상태의 큰 말풍선은 제거하고, 선택한 구/시 뱃지 자체를 크게 강조해 중복 정보와 지도 가림을 줄였다.
- 확대 기준점을 광역 중심이 아니라 세부 구/시 뱃지 범위 중심으로 바꾸고, 작은 광역시는 줌 배율을 더 올려 모바일 터치 타겟을 키웠다.
- 확대 상태에서는 선택하지 않은 주변 지역의 면/경계 투명도를 크게 낮춰 서울 선택 시 경기도 같은 주변 지도 정보가 시야를 덜 차지하게 했다.
- 전국 지도 광역 뱃지 자체를 클릭/호버 가능한 이벤트 레이어로 바꿔, 서울처럼 면적이 작고 주변 지역에 둘러싸인 곳도 뱃지로 정확히 선택되게 했다.
- 세부 구/시 선택은 지도 안 작은 뱃지 클릭 방식에서 지도 아래 큰 스크롤 리스트 방식으로 바꿨다.
- 세부 리스트는 mock 티오 진행률을 `3/5`, `2석 남음` 형태로 보여줘 모바일에서도 손가락으로 안정적으로 선택할 수 있게 했다.
- 멤버십 지역 첫 단계의 하단 CTA는 `다음` 대신 `내 위치 불러오기`로 바꾸고, GPS 성공 시 reverse-geocode 결과로 광역 지역을 자동 줌인한다.
- 위치 권한 거부/실패 시 온보딩의 주소 검색 API를 재사용해 같은 화면에서 수동 동네 검색 후 지역 티오로 진입하게 했다.
- 세부 티오 리스트 상단의 선택 지역 헤더는 아래 행과 정보가 중복돼 제거했다.
- 세부 티오 행은 얇은 3열 좌석판 형태로 바꿔 한 모바일 화면에 더 많은 구/시가 보이게 했다.
- GPS/수동 검색으로 잡힌 세부 구/시는 리스트 맨 위로 올려 사용자가 내 위치가 선택됐는지 바로 확인하게 했다.
- 전국보기에서는 시/군/구 내부 상세 경계선을 숨기고, 광역시/도 선택 후 확대 상태에서만 세부 경계선을 표시한다.
- 내 위치로 잡힌 세부 구/시만 리스트 맨 위에 고정하고, 사용자가 다른 구/시를 선택해도 리스트 순서는 더 이상 바뀌지 않게 했다.
- 선택된 세부 구/시 행은 라이트/다크모드 모두에서 보이도록 굵은 border/ring으로 강조하고, 내 위치 배지를 별도로 표시한다.
- 남은 좌석이 절반 이하이면 노랑, 1~2석이면 빨강으로 좌석 수와 진행바 색을 바꿔 마감 임박감을 강화했다.
- 전국보기의 `seat map` 안내 박스는 제거하고, 모바일 첫 화면 헤드라인과 지도 영역을 더 크게 쓴다.
- 첫 단계 하단 CTA는 이전 버튼 placeholder를 제거해 `내 위치 불러오기` 버튼이 좌우 균등하게 꽉 차도록 했다.
- 지역/세부 구 선택 다음 단계에 내 동네 샘플 매물 화면을 추가했다.
- 샘플 화면은 기본 카테고리 워터마크 이미지, 예상 차익, 평균 판매일, 남은 자리 수를 한 화면에 보여줘 지역 선택 직후 가치와 FOMO를 직관적으로 전달한다.
- 멤버십 신청 전 흐름은 `지역 티오 선택 → 내 동네 샘플 → 공개 제한 이유 → 신청` 순서로 정리했다.
- 샘플 매물은 프론트 mock 배열이 아니라 `/api/membership/local-sample`에서 실제 `mvp_candidate_pool.status=ready` + `mvp_raw_listings.source=daangn` + `listing_state=active` 후보를 읽어 보여주도록 바꿨다.
- 신청 전에는 원본 링크와 실매물 사진을 열지 않고, 피드와 같은 SKU 기본 이미지/매입가/시세/예상 차익/판매속도 표본만 노출한다.
- 샘플 API의 request-time live join은 느려서 폐기하고 `mvp_membership_local_samples` DB cache를 추가했다.
- `/api/cron/membership-local-samples`가 실제 당근 ready/active 후보를 구/지역별로 미리 구워 저장하고, 신청 화면은 캐시 row 1개만 읽는다.
- 샘플 카드는 피드 카드와 맞춰 실사진, 제목, `+예상 차익`, `+%`, 매입가, 시세, 당근 지역/판매속도 chip을 한 카드 안에 배치했다.
- 운영 DB에 캐시 테이블을 적용했고, 초기 refresh 결과 131개 district sample이 생성됐다.

## 보류

- 실제 인구/지역별 신청자 기반 티오 산정은 아직 mock 수치 기반이다.
- 지도 클러즈업 애니메이션의 세밀한 easing/gesture 조정은 다음 디자인 라운드에서 더 다듬는다.
- 선택 district 캐시가 없을 때는 최신 캐시 row 하나로 fallback한다. 향후 지역별 표본이 충분해지면 모든 선택 구/시를 100% exact sample로 채운다.
