## 보강 내용 — `tick-pipeline.ts:1750` 옵션 A regex 확장

### 1. NEGATION_INTENSIFIER 강조어 확장
기존: `일절|전혀|아예|단\s*하나|일체|진짜`
확장: 위 + `진심|정말|완전|완벽|단연|하나도|결코|거의|딱히|별로|한\s*번도|단\s*한\s*번도`

### 2. NEGATION_KEYWORDS 키워드 확장
기존: `하자|손상|수리|교체|고장|불량|파손|깨짐|기스|스크래치|찍힘`
확장: 위 + `크랙|찌그러짐|얼룩|오염|침수|낙상|충격|떨어(?:뜨|트)림|이상|문제|사고|결함|이력|흠집`

### 3. 패턴 3개로 분리
- **패턴 1**: keyword + (된/받은/적 등 동작형 어미) + particle + 강조어 + 없
- **패턴 2**: 강조어 + keyword + particle + 없 (한국어 어순 자유)
- **패턴 3**: 동작형 부정 (떨어뜨린/충격받은/침수된/낙상된/박살난/밟힌/튕긴) + (적/일/경험/이력) + 강조어 + 없

### 4. damage keyword 확장
"떨어(?:뜨|트)린|떨어(?:뜨|트)림|기스 있|얼룩 심|얼룩 있|이력 있|문제 있|결함 있|수리 이력|교체 이력" 추가.

## lesson learned (사용자 정직 검증의 가치)

내가 wave256-implementation.md 박을 때 "옵션 A 만으로 이 매물 잡힘" 라고 보고. 그러나:
- 사용자 매물 (pid 405343339) 만 test 했음 (`하자 일절 없` 패턴)
- **다른 한국어 변형 test 안 함** — "기스 진심 없" / "떨어뜨려서 충격 전혀없" 등
- → frame 작동 OK 라고 추정. **사용자 직접 검증 안 했으면 안 잡힘**

→ **사용자 정직 검증 (기출 변형 직접 질문) 이 더 강력한 test 그 자체**.

미래 wave 에 박을 때:
1. user-reported case 만 test 박지 말기. **변형 안전 패턴 3-5개 추가** (intensifier 다양화, 어순 변화).
2. 변형 안 잡히면 root regex 보강.
3. lesson: regex 자신감 과잉 차단 — user 한 사람의 검증 변형이 frame 의 진실성을 드러냄.
