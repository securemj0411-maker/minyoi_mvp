# 2026-05-21 Server Side Free Feed Masking

## 결정
- 무료 피드의 원본 `pid`, 원제목, 원본 이미지, 정확한 가격/차익은 서버 응답 단계에서 마스킹한다.
- 무료 피드에는 암호화된 `accessToken`과 가짜 pid만 내려주고, 상세보기 권한 확인 API에서만 토큰을 복호화해 실제 pid를 사용한다.
- `/api/packs/pool/analysis`는 상세보기 접근 기록이 있는 pid만 허용한다.

## 이유
- 프론트에서만 가리면 개발자도구 Network 응답에 원본 pid/name/image가 남아 우회가 가능하다.
- 번개장터 URL은 pid만 알아도 직접 접근할 수 있으므로, 무료 피드에서는 pid 자체도 숨겨야 한다.

## 보류
- 캡처, 기억, 외부 공유처럼 사용자가 이미 상세보기에서 합법적으로 본 정보의 2차 유출은 기술적으로 완전히 막지 않는다.
- 크레딧 보유자의 피드 원본 노출 정책은 현 상태를 유지한다. 결제 전환과 남용 신호를 보고 별도 조정한다.
