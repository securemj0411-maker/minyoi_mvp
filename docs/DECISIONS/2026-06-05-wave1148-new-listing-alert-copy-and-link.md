# 2026-06-05 Wave 1148 — 새매물 알림 화면/문구 정리

## 결정
- `새매물 알림` 화면은 알림 설정 화면으로 다룬다.
- 일반 진입 시 `받은 알림` 목록을 보여주지 않는다.
- 텔레그램 딥링크처럼 `pid`가 있는 진입에서만 해당 알림 매물 확인 카드를 보여준다.
- 사용자 노출 문구에서 `핫딜` 표현을 제거하고 `새매물 알림`으로 통일한다.
- 알림 설명은 `차익이 큰 새 매물`이 아니라 `내 동네에 차익 나는 새 매물` 기준으로 바꾼다.

## 구현
- `src/components/hotdeal-alerts-view.tsx`
  - 일반 알림 설정 화면에서 `받은 알림` 섹션을 숨겼다.
  - `pid` query가 있을 때만 `알림 매물 확인` 섹션을 렌더한다.
  - 설명과 동작 방식 카피를 지역 기반 새매물 알림 톤으로 정리했다.
- `src/components/hotdeal-reservations.tsx`
  - 사용자 노출 `핫딜` 표현을 `새매물 알림` 또는 `알림`으로 변경했다.
  - opened 카드의 액션을 `번장에서 거래`에서 `원문 열기`로 변경했다.
- `src/app/api/me/hotdeal/reservations/route.ts`
  - 원문 링크를 번개장터 pid로 하드코딩하지 않고 `mvp_listings.url`을 내려주도록 변경했다.
- `src/lib/hotdeal.ts`
  - 텔레그램 알림 제목과 버튼 문구를 `새매물 알림` 톤으로 변경했다.

## 보류
- 텔레그램 링크가 곧바로 피드 상세 모달을 여는 딥링크 구조는 별도 작업으로 남겼다.
- 현재는 기존 reservation/open 흐름을 유지하면서 알림 매물 확인 화면으로 들어온다.
