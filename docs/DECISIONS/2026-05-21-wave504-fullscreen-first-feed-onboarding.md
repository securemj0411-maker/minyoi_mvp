# Wave 504 — 첫 피드 온보딩 풀스크린화와 무료 상세 라벨 수정

## 문제
- 첫 가입 hook이 피드 안 카드로 들어가 있어 토스식 인앱 첫 화면 느낌이 아니었다.
- 통계/예산/CTA가 한 덩어리 카드에 들어가 정보가 뭉쳤다.
- 예산 `15만원 이하` 선택 시 후보가 1~3개로 줄어 첫 피드가 빈약해 보였다.
- 무료 상세 3회가 남아있는데도 피드 카드가 `잠김`처럼 보여, 클릭하면 열리는 UX 모순이 있었다.

## 결정
- 첫 방문 hook은 피드 안 카드가 아니라 화면 전체를 덮는 2-step 인앱 온보딩으로 보여준다.
- 1단계는 “오늘 볼 만한 후보만 남김”과 필터링 숫자, 2단계는 예산 선택으로 분리한다.
- 예산 필터는 후보가 충분할 때만 적용한다. `MIN_BUDGET_FILTER_RESULTS = 6` 미만이면 전체 후보를 유지해 첫 피드가 죽지 않게 한다.
- 피드가 마스킹되어도 무료 상세가 남아있으면 `무료 상세 가능`, `무료 상세 열기`로 표시한다. 무료 소진 후에는 기존 잠금/크레딧 문맥으로 간다.

## 구현
- `/api/packs/pool` 응답에 `detailAccess` 스냅샷을 추가했다.
  - `freeUsed`, `freeLimit`, `creditBalance`
- `src/lib/detail-access.ts`에 `getDetailAccessSnapshot`을 추가했다.
- `ExploreClient` 첫 방문 UI를 fixed fullscreen 2-step 온보딩으로 교체했다.
- 클라이언트 예산 필터가 너무 적은 결과를 만들면 fallback으로 전체 후보를 보여준다.
- 마스킹 카드 문구를 무료 상세 잔여 횟수에 맞춰 다르게 보여준다.

## 보류
- 예산 선택을 실제 개인화 ranking으로 반영하는 작업은 추후 진행한다. 지금은 pool 희소성이 더 큰 문제라 hard filter로 쓰면 안 된다.
- 첫 방문 온보딩의 숫자 애니메이션/전환 애니메이션은 MVP 이후 추가한다.
