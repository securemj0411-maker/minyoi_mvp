# 2026-06-06 Wave 1177 — 피드 초기 체감 속도 스냅샷

## 결정
- 추천 피드가 마지막으로 성공한 동일 필터 결과를 2분 동안 localStorage snapshot 으로 보관한다.
- 같은 사용자/필터로 다시 들어오면 snapshot 을 즉시 화면에 보여주고, 서버 최신 피드는 기존처럼 `cache: no-store` 로 다시 받아 교체/보강한다.
- snapshot 은 최대 80개만 저장한다. 당근 실제 보강 요청은 별도로 500개까지 유지한다.

## 이유
- 초반 네트워크/DB 후처리 시간 동안 빈 로딩 화면만 보이면 서비스가 느리게 느껴진다.
- 첫 방문의 실제 API 시간은 줄이지 못해도, 상세에서 돌아오기/필터 왕복/재방문에서는 즉시 매물 카드가 보이는 편이 체감이 낫다.
- stale 데이터를 오래 보여주면 신뢰가 깨질 수 있어 TTL 을 짧게 잡고, 최신 fetch 는 항상 다시 수행한다.

## 보류
- DB 선계산 feed snapshot 테이블이나 streaming 응답은 이번에 도입하지 않았다.
- 첫 방문 TTFB 자체를 줄이려면 `/api/packs/pool`의 시세/velocity join 경로를 별도 fast lane 으로 분리해야 한다.
