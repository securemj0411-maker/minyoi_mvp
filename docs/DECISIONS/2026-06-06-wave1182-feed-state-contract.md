# 2026-06-06 Wave 1182 — 피드 상태 계약 도입

## 결정

- `/api/packs/pool` 응답에 `feedState`를 추가했다.
- `feedState`는 응답이 캐시/빠른 조회/전체 조회인지, 결과가 부분 응답/완료/진짜 빈 상태인지, 프론트가 이어서 조회해야 하는지를 명시한다.
- 프론트의 초기 보강 조회와 빈 상태 보강 조회는 더 이상 예산/당근 여부만 보고 추측하지 않고 `feedState.shouldRequestContinuation`을 기준으로 동작한다.

## 이유

- 기존 구조는 피드 API가 “이 응답은 임시 빠른 응답”인지 “전체 확인 후 진짜 없음”인지를 말하지 않았다.
- 그래서 프론트가 `items.length`, 예산 필터, 당근 거리 정렬 여부로 추측했고, 이 때문에 빈 화면이 떴다가 새로고침하면 매물이 보이는 문제가 반복됐다.
- 광고 전에는 이런 조건 덧대기식 보정이 아니라 서버-프론트 계약을 고정해야 같은 문제가 다른 화면에서 재발하지 않는다.

## 보류

- 지역별 추천 피드를 DB에 사전 계산해 저장하는 구조는 별도 작업으로 남긴다.
- 이번 변경은 현재 API 위에 상태 계약을 추가해 즉시 재발 위험을 줄이는 작업이며, 장기적으로는 지역/예산/출처별 feed card snapshot 테이블을 두는 것이 더 빠르고 안정적이다.
