# 2026-05-20 Wave 417 — 상세페이지 초보자 가이드

## 결정
- 상세페이지 첫 진입 시 기존 밀도 높은 분석 화면을 바로 보여주기보다, 초보자용 단계 가이드를 먼저 보여준다.
- 가이드는 `판매자 신뢰 → 비교 매물 → 시세 흐름 → 매입가 → 되팔 때 비용 → 안전결제 → 되팔 곳 → 판매 속도 → 마무리` 흐름으로 구성한다.
- 각 단계는 기존 `RevealCard`에 이미 들어있는 셀러 평점/후기 수, 시세 기준, 배송비·수수료·안전버퍼 계산, 판매 속도, 예상 순익 데이터를 재사용한다.
- 사용자는 `다음`으로 단계별 확인 후 `상세 분석 보기`로 기존 화면에 들어갈 수 있다.
- 싫은 사용자는 `초보자 가이드 스킵하기`를 눌러 해당 매물에서는 기존 상세 분석을 바로 볼 수 있다.
- 스킵은 매물 단위로 저장한다. 다만 여러 매물에서 4회 이상 연속적으로 스킵하면 “이 사용자는 자동 안내를 선호하지 않음”으로 보고 기본 자동 노출만 끈다.
- 자동 노출이 꺼져도 상세 화면의 `쉽게 보기` 버튼으로 초보자 가이드를 다시 열 수 있다.
- 사용자 피드백 반영: 가이드는 카드형 모달이 아니라 상세 모달 전체를 덮는 인앱 화면으로 보이며, 가이드 중에는 관련 매물·원본 매물 버튼·기존 상세 덩어리를 숨긴다.
- 첫 단계에는 상품 이미지를 보여주되 모바일 높이를 과하게 쓰지 않도록 줄이고, 뒤로가기 버튼은 사진 위에 겹칠 수 있게 둔다.
- 비교 매물은 초보자에게 가장 중요한 근거이므로 시세 그래프보다 먼저 보여주고, 실제 비교 매물 샘플을 별도 단계로 노출한다.
- 시세 그래프는 비교 매물과 분리해 “비슷한 상태 시세가 어떻게 움직였는지”만 한 화면에서 본다.
- 비용 계산은 한 화면에 몰아넣지 않고 `매입가(상품가+내가 낼 배송비)`와 `되팔 때 비용(안전결제 수수료+재배송비+안전버퍼)`으로 나눈다.
- 안전결제 단계는 보장형 환불 문구를 피하고, 에스크로 구조·구매확정 전 확인·문제 발생 시 확정 보류 중심으로 설명한다.
- 되팔 곳 단계는 번개장터 재판매와 당근 직거래 예상 차익을 비교하되, 당근은 지역/네고 부담을 같이 표시한다.
- 문구는 지표 나열이 아니라 “이 상품 판매자는 후기가 n건이고 평점이 n점...”처럼 초보자가 바로 이해할 수 있는 문장으로 쓴다.

## 변경
- `pack-reveal-modal.tsx`에 `BeginnerGuideWalkthrough`를 추가했다.
- 초보자 가이드는 기본 3회까지만 자동 노출하고, 본 매물 pid와 스킵 횟수는 localStorage에 저장한다.
- 가격/순익 관련 카피는 보장형 표현을 피하고 “관측”, “기준”, “달라질 수 있음” 중심으로 작성했다.
- `RevealCardItem` 내부 가이드 렌더링을 제거하고, `PackRevealModal` 단에서 `guideModeActive`로 성공 화면 전체를 전환한다.
- 단계 전환 시 작은 fade/slide 애니메이션을 추가하고, 다음 단계 진입 시 스크롤을 맨 위로 되돌린다.
- 상단 두꺼운 진행바는 모바일 정보를 가리므로 제거하고, 작은 뒤로가기 버튼과 `n/전체` 카운터만 남긴다.
- 마지막 마무리 단계는 앞에서 이미 설명한 매입가/수수료/차익 계산을 반복하지 않고, 확인한 근거 체크리스트와 상세 분석 진입만 담당한다.
- 기존 상세 화면 첫 영역에 `쉽게 보기` 버튼을 추가해 수동 재진입 경로를 둔다.
- 판매자 평점/후기 수는 문장 속 숫자로 묻히지 않도록 별 SVG, 후기 배지, 굵은 숫자 카드로 강조한다.

## 보류
- 사용자 계정 단위의 초보자 모드 설정 저장은 아직 하지 않는다. 현재는 브라우저 localStorage 기준이다.
- A/B 테스트와 계정 설정 UI는 후속 UX 다듬기에서 본다.
