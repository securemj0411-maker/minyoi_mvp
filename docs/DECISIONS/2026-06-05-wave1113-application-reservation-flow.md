# 2026-06-05 Wave 1113 — 멤버십 신청 예약 흐름 정리

## 결정

- 신청 플로우 안의 10분 업셀 모달은 제거한다. 업셀은 피드 내 멤버 전용 제안으로만 유지한다.
- `지금 바로 자리 차지하기` 이후에는 작은 닫기 모달이 아니라, 토스식 단계 화면 안에서 기간 선택을 바로 보여준다.
- 기간 선택 후에는 `지역 자리 1석 확보` 화면으로 전환하고, 7분 입금 카운트다운을 노출한다.
- 계좌 안내는 자리 확보 화면 안에 두고, 입금 후 `입금했어요` 버튼을 누르면 기존 5분 승인 확인 흐름으로 이어진다.

## 구현

- `MembershipApplicationClient`에서 신청 흐름 업셀 토큰/모달 상태와 JSX를 제거했다.
- `autoOpenSelector`는 모달 자동 오픈이 아니라 인라인 기간 선택 화면 렌더링으로 변경했다.
- 예약 성공 시 7분 예약 만료 타이머를 생성하고, 예약 완료 화면에 남은 시간을 표시한다.
- `PlansApplicationFlow`에서 선택 지역/샘플 지역명을 신청 컴포넌트에 전달해 `상도동 자리 1석` 같은 문구를 만들 수 있게 했다.
- `/plans`, `/api/membership/apply`, `/api/membership/status`, `/api/membership/deposit-notify`에서 입금 확인 전 7분이 지난 pending 신청을 `rejected`로 만료 처리한다.

## 보류

- 별도 예약 만료 컬럼/cron은 추가하지 않았다. 현재는 기존 `created_at`, `deposit_confirmed_at`, `status`만으로 서버 요청 시 만료를 정리한다.
