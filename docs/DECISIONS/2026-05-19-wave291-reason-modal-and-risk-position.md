# 2026-05-19 Wave 291 — 추천 이유 모달화와 안전 모달 위치 보정

## 배경
- `/me` 상세에서 `왜 이 제품이 안전한가요?`를 눌렀을 때 위험 신호 모달이 모바일 화면에서 너무 아래에 떠서 하단 고정 CTA와 겹쳐 보였다.
- `왜 이걸 추천했나요?`는 카드 안에서 펼쳐지는 `<details>` 구조라서 본문 높이를 밀어내고, 그래프와 주요 가격 정보를 한눈에 보기 어렵게 만들었다.

## 결정
- 위험 신호 모달의 모바일 기준 위치를 `top-[45%]`에서 `top-[39%]`로 올리고, 본문 최대 높이도 함께 줄여 하단 CTA 침범을 줄였다.
- 추천 이유 패널은 목록 안에서는 얇은 요약 카드로 유지하고, `근거 보기`를 누르면 별도 dialog로 띄우도록 바꿨다.
- 추천 이유 dialog에는 기존의 좋은 점, 확인할 점, 계산 기준, 표본/판매완료/신선도 정보를 유지한다. 기능은 줄이지 않고 표시 방식을 바꾼다.

## 보류
- 추천 이유 dialog를 라우트 기반 상세 페이지로 분리하는 작업은 보류한다. 현재 상세 페이지 안 보조 설명이므로 lightweight dialog가 더 빠르고 영향 범위가 작다.
- 안전 모달과 추천 이유 모달의 공통 dialog 컴포넌트 추출은 추후 반복이 더 늘어날 때 검토한다.

## 검증 계획
- `/me` 계약 테스트에서 추천 이유가 dialog 패턴을 갖는지 확인한다.
- 위험 신호 계약 테스트에서 새 위치와 높이 기준을 확인한다.
- 기존 build 실패가 unrelated dirty worktree 문제인지 분리해서 기록한다.
