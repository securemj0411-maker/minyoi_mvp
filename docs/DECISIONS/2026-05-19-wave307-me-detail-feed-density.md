# 2026-05-19 Wave 307 — /me detail feed density

## Context

사용자가 `/me` 상품 상세에서 디버깅용 정보와 반복 카드 UI가 사용자 판단을 방해한다고 지적했다.

- `검증 메모 · 추천 평가`는 운영자/디버깅용 문구라 사용자 상세에 노출할 이유가 없다.
- `상품 N건 검증 → N건 추천`, `AI 기반 시세 추천 — 수익 보장 X`는 법적/안내 성격은 있지만 첫 화면을 밀 필요가 없다.
- `내 다른 추천 매물`은 수익만 보여줘서 `/me` 피드에서 보던 매입가, 시세, 상태 등급 맥락이 빠진다.
- 목록이 카드 안 카드처럼 보여 당근마켓식 피드 밀도와 멀다.

## Decisions

1. 상품 상세에서 `검증 메모 · 추천 평가` 영역을 제거했다.
   - 운영자 메모/품질평가는 사용자 화면이 아니라 admin/debug 또는 신고 플로우에 둔다.

2. 일반 유의사항은 상세 최하단 접힘 영역으로 이동했다.
   - `시세 추천 유의사항` summary 아래에 검증 건수, 전체 본품 비교, 수익 보장 아님 문구를 둔다.
   - 사용자 첫 판단 영역과 다른 추천 매물 CTA를 밀지 않게 했다.

3. `내 다른 추천 매물`을 카드형 묶음에서 피드형 리스트로 바꿨다.
   - 각 row에 매입가, 시세, 시세 source badge, condition photo badge, condition label을 노출한다.
   - section 자체의 둥근 카드 프레임과 row 카드 프레임을 제거하고 border/divider 중심으로 정리했다.
