# 2026-06-06 wave1169 - 멤버십 관리/기간 선택 UI 정리

## 결정
- 이미 멤버십이 활성화된 사용자의 `/plans` 화면은 신규 신청/설득 페이지처럼 2열 카드로 나누지 않는다.
- `내 멤버십` 단일 카드 안에서 활성 상태, 남은 기간, 만료일, 연장 적용 방식, 연장 CTA를 한 흐름으로 보여준다.
- 신규 신청과 연장 모두 기간 선택은 별도 미니 모달이 아니라 어두운 전체형 선택 모달로 통일한다.
- 연장 예약/입금 로직은 기존 `MembershipApplicationClient`를 유지하고, 선택 UI만 바꾼다.

## 구현
- `src/app/plans/page.tsx`의 활성 회원 레이아웃을 `lg:grid-cols` 2열 구조에서 단일 `max-w-[520px]` 카드로 변경했다.
- 기존 우측 `Renewal` 설명 카드는 제거하고, 연장 버튼은 단일 카드 하단 액션 영역으로 이동했다.
- `남은 기간을 확인하고...` 보조 설명과 `premium pass / ACTIVE / M` 장식성 뱃지는 제거했다.
- PC에서 카드가 여전히 히어로처럼 보이지 않도록 최대 폭을 520px까지 줄이고, 화면 제목은 멤버십 카드 내부의 관리 헤더/상품명으로 압축했다.
- 남은 기간을 핵심 숫자로 두고, 만료일과 연장 방식은 같은 줄의 보조 메타 정보로 압축했다.
- 최종적으로는 일반적인 구독/멤버십 관리 화면처럼 상단 헤더, 활성 상태 배지, 핵심 잔여 기간 카드, 만료일/연장 방식 보조 카드로 재구성했다.
- 사용자 레퍼런스에 맞춰 로고 중심의 원형 진행 링, 남은 기간 진행바, 가입/잔여율/만료일 메타를 가진 멤버십 패스형 카드로 다시 구성했다.
- 로고 뒤에 별도 파란 원을 깔지 않고 `logo.svg` 자체를 원형으로 마스킹하도록 수정했다.
- 카드 최대 폭, 아이콘 크기, 여백, 보조 카드 수를 줄여 PC에서 한눈에 들어오는 미니멀 멤버십 카드로 압축했다.
- `src/components/membership-application-client.tsx`의 기간 선택 모달을 레퍼런스처럼 어두운 패널, 상단 MEMBERSHIP 헤더, 2x2 요금 카드, 하단 닫기/예약 CTA 구조로 바꿨다.
- 선택된 기간 카드는 파란 테두리, 어두운 블루 배경, `선택됨` 배지로 강조하고 나머지는 낮은 대비의 다크 카드로 정리했다.
- 모바일/PC 모두 같은 컴포넌트로 신규 신청과 연장 신청을 처리하되, 문구만 `자리 예약하기`와 `연장 예약하기`로 갈라지게 했다.
- PC에서 모달이 랜딩 섹션처럼 커 보이는 문제가 있어 최대 폭을 720px로 줄이고, 카드 높이/제목/CTA/여백을 함께 압축했다.
- 모바일에서는 플랜 4개와 CTA가 한 화면에 들어오도록 플랜 그리드를 2열 고정으로 바꾸고, 카드 보조문구는 PC에서만 보이게 줄였다.
- 3개월 99,000원 플랜은 단순 `추천` 대신 `인기 1위`, `가장 많이 선택한 시작 구간`으로 바꿔 사회적 증명 신호를 더 명확하게 했다.

## 보류
- 토스/계좌 송금 단계의 상세 입금 확인 UI는 이번 변경 범위에서 제외했다.
- 비회원/신규 신청 온보딩 플로우는 유지했다.
