# 2026-06-04 Wave 1046 — membership seat reservation and bank transfer

## Trigger

멤버십 신청 퍼널을 단순 `운영자 검토 대기`가 아니라, 선공개 300명 기준의 `자리 예약 후 계좌이체` 흐름으로 전환하기로 했다.

사용자 결정으로 지역별 티오 확인은 현재 mock 처리하며, 신청자는 모두 `신청 가능`으로 통과시킨다.

## Decisions

- `/plans`는 선공개 300명 티오와 현재 예약 수를 보여준다.
- 지역 티오는 mock으로 항상 `신청 가능`으로 보여준다.
- 플랜 선택 후 신청 상태는 `입금 대기`로 표현한다.
- PG 결제는 쓰지 않고 계좌이체 안내를 보여준다.
- 운영자 검토 프리미엄은 `입금 확인 후 운영자 승인`으로 남긴다.
- 승인 기간은 기존 90일 고정이 아니라 선택한 상품의 개월 수를 따른다.

## Applied

- `/plans` 상단 멤버십 note에 선공개 티오와 지역 티오 상태를 추가했다.
- 신청 완료 UI를 `자리 예약 완료 · 입금 대기`로 변경하고 계좌 정보를 노출했다.
- 멤버십 신청 텔레그램 알림을 `자리 예약 / 입금 대기`로 변경했다.
- cau 운영자 멤버십 신청 패널을 입금 확인 중심 문구로 변경했다.
- 관리자 승인 API가 `product_key`의 `months`를 읽어 `current_period_end`를 계산하도록 변경했다.

## Deferred

- 실제 지역별 quota 계산/차단은 아직 구현하지 않는다.
- 계좌이체 자동 입금 확인은 아직 없다. 현재는 운영자 수동 승인이다.
- 토스/카카오페이 송금 deep link를 멤버십 신청 완료 화면에 붙이는 작업은 다음 wave로 미룬다.
