# Wave 1197 — 입금 모달 신뢰 배지 (계좌입금 공포 차단)

날짜: 2026-06-06
관련: membership-application-client.tsx, payment-trust-card.tsx, Wave 1196
owner 요청: "전환율 / FOMO / 계좌입금 공포 / 위계 개선할 거 보이니?"

## 분석 (입금 모달 4관점)

이미 잘 된 것:
- 입금 금액 22px 파란 강조 / 계좌 복사 버튼 / 토스 deep link
- 카운트다운(자리 만료) + 선착순 티오 = FOMO 충분
- 입금했어요 CTA 강조 (Wave 1196)
- 사업자 정보(PaymentTrustCard) 존재

**최대 누수 = 계좌입금 공포 구간**:
- PaymentTrustCard 의 핵심 신뢰 신호(사업자번호·예금주 실명 일치·환불정책)가
  `<details>`로 **접혀** 있음. "사업자 정보 확인 ⌄"을 눌러야만 보임.
- 타이밍 문제: 계좌이체 직전(처음 보는 사이트에 송금 = 공포 최고조)에 정작 안심 정보가
  클릭 뒤에 숨음 → 대부분 안 누르고 이탈.

## 변경 (P0 — 신뢰 배지 상시 노출)

입금했어요 버튼 **바로 아래**에 배지 한 줄 (showPaymentDetails && 입금 전):
```
✓ 사업자 등록업체   ✓ 예금주 실명 일치   ✓ 미승인 시 전액 환불
```
- 누르기 직전 ✓ 3개가 항상 보임 → 공포 즉시 상쇄.
- "미승인 시 전액 환불"은 emerald 강조 — 손실 회피(돈 떼일 공포) 정면 차단.
  **환불정책 1조**("입금 후 멤버십 미활성화 시 전액 환불")에 근거 → 과장 아님.
- PaymentTrustCard(상세: 사업자조회 링크/사업자번호/환불정책)는 그대로 아래 유지 = 보완.

P1(예금주=사업자 대표 연결)은 배지 "예금주 실명 일치" + trust card 상세로 통합.

## 의도적으로 안 한 것

- **FOMO 추가 안 함**: 카운트다운 + 선착순이 이미 충분. 더 세게 하면 "급하게 돈 보내라"
  압박처럼 보여 오히려 공포 증폭. 지금이 균형점.
- **과장 사회적 증거 안 함**: 일반인 친화 원칙(과장 금지). 실데이터 없는 "N명 이용 중" 류 배제.
