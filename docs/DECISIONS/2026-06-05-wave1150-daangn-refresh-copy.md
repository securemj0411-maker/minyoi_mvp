# 2026-06-05 Wave 1150 — 당근 피드 하단 CTA를 새로고침으로 변경

## 결정
- 당근/가까운순 피드에서는 `더 찾아보기` 표현을 쓰지 않는다.
- 당근 모드의 하단 CTA는 `근처 매물 새로고침`으로 바꾸고, 동작도 append가 아니라 현재 위치 기준 재조회로 바꾼다.
- 중고나라/번개장터까지 확장한 전체 피드에서는 기존처럼 `더 찾아보기`를 유지한다.

## 원인
- `/api/packs/pool`은 전체 ready rows를 전부 내려주지 않고 `PAGE_SIZE=30`으로 잘라 준다.
- refresh 요청은 기존 pid를 제외하고 추가 후보를 받을 수 있지만, 당근/근처 피드에서는 사용자가 “내 근처 매물은 다 보여줘야 하는 것 아닌가”라고 느끼기 쉽다.
- 그래서 당근 모드에서 `더 찾아보기`는 새 후보가 숨겨져 있다는 인상을 줘 UX상 부정확했다.

## 구현
- `src/components/explore-client.tsx`
  - 당근 focused view용 `refreshNearbyDaangnFeed()`를 추가했다.
  - 당근 모드 CTA 문구를 `근처 매물 새로고침`으로 변경했다.
  - 당근 모드에서 버튼 클릭 시 `loadPool(false, { serverSource: "daangn", serverSort: "distance" })`로 현재 후보를 재조회한다.
  - 피드 끝 안내 카피도 당근 모드와 전체 피드 모드를 분리했다.

## 보류
- 서버가 지역별 전체 당근 ready 후보 수를 계산해서 “현재 N개 전부 표시”처럼 명확히 내려주는 API 확장은 별도 작업으로 남겼다.
