# 2026-05-17 welcome 로딩 UX fix — 큰 스켈레톤 + 가시적 안내

## 사용자 보고

- 시간: 2026-05-17
- 보고: "스켈레톤 보였다가 갑자기 추천 매물을 준비하고 있어요. 잠시만 기다려주세요 이거 개 작은 글씨로 거의 사용자 안보일정도로 갑자기 스켈레톤 없어지고 저 작은거 보이니까 사용자 당황할듯??"
- 요구: "그냥 나오기 전까지 쭉 스켈레톤을 하던가 아니면 진짜 가시적으로 뭔가를 하던가"
- 동작 자체: 4개 정상 노출 (새로고침 X) — 무한 로딩 fix 작동. UX 만 별로.

## 원인

이전 fix 의 welcomePending 빈 메시지가 작은 1줄 (`text-xs`) 회색 박스. 페이지 로딩 스켈레톤 (큰 카드 grid) → 갑자기 작은 한 줄로 줄어들면서 "끝났나? 뭐지?" 인지 단절. 사용자 4 카드 결과까지 대기 시간 사이에 빈 인상.

## 변경

- 시간: 2026-05-17
- 파일: [src/components/user-reveal-dashboard.tsx](../../src/components/user-reveal-dashboard.tsx)
- 위치: empty state 분기 (L996 부근)
- 변경 전: 1줄 회색 박스 `text-xs`. query/welcomePending/없음 모두 같은 컨테이너.
- 변경 후: 분기 분리, welcomePending 일 때 아래로 교체.
  - **안내 박스**: 큰 글씨 (`text-base sm:text-lg font-black`) "가입 환영 매물 준비 중" + spinner svg (animate-spin) + 보조 설명 "첫 추천 매물 4개를 골라서 가져오고 있어요"
  - **스켈레톤 grid**: 4 카드 `h-48 animate-pulse` (페이지 로딩 스켈레톤과 동일 톤 — `bg-[#f1eadf]`)
- 다른 분기 (검색결과 없음 / 매물 없음) 는 기존 작은 박스 유지.

```tsx

  ) : (
    /* 작은 회색 박스 "아직 본 추천 상품이 없습니다" */

```

## 검증

- `npx tsc --noEmit` — 변경 파일 에러 0.
- 시나리오 (수동):
  - 신규 가입 → /me 진입 → 페이지 스켈레톤 (loading=true) → user 로드 후 사이드바/검색바 노출 + 콘텐츠 영역은 큰 "가입 환영 매물 준비 중" 박스 + 4 스켈레톤 카드 → welcome 완료 → 4 실제 카드 swap-in.
- 사용자 측 prod 배포 후 가시성 확인 필요.

## 위험

- 스켈레톤 박스 다크모드 색상 `bg-zinc-800/60` 으로 통일. 라이트모드 `#f1eadf` 는 페이지 로딩 스켈레톤 매치.
- welcomePending 빈 상태가 query 와 분기 분리 — code 길이 ↑ but 명확.
- 안내 카피 "4개" 하드코드 — 향후 WELCOME_CARDS 변경 시 코드 같이 수정 필요. 현재 정책 4 고정이라 OK.

## Lesson

비동기 대기 UX 는 "안내 톤" 이 인지 단절 막는 핵심. 페이지 스켈레톤 (큰 영역) → 작은 한 줄 텍스트 = 시각 전이 broken. 동일 시각 비중 유지 (큰 카드 grid 스켈레톤 + 큰 안내 박스) 가 자연스러움.
