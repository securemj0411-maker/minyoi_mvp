# Wave 252 — 더 찾아보기 초회 설문과 고급 검색 분리

## 결정
- `/me`의 `더 찾아보기` 모달에서 `회원님 보호` 차단 통계와 온보딩 배너를 제거했다.
- 추천 조건 UI는 초회 1회 설문으로 바꿨다.
  - 매입 가능한 최대 예산
  - 선호 스타일: 빠른 회전 / 균형 / 수익 우선
- 설문 저장 후에는 기본 화면에서 추천 상품 수만 고르게 한다.
- 기존 세부 조건(프로필, 매입가, 신선도, 차익, 신뢰도, 카테고리, 계산식)은 `고급 검색`을 눌렀을 때만 보이게 했다.

## 이유
- 사용자가 `더 찾아보기`를 누른 순간의 목표는 통계 확인이 아니라 새 추천을 여는 것이다.
- 리셀 초보/준리셀러에게 슬라이더와 세부 조건을 먼저 보여주면 의사결정 부담이 커진다.
- 플랫폼 평가에서 나온 개인화 방향은 “복잡한 검색 UI”가 아니라 “초기 질문으로 기본값을 자동 조정”하는 방식이 더 맞다.

## 보류
- 설문값은 우선 브라우저 localStorage에 저장한다. 서버 프로필 저장과 추천 품질 분석용 이벤트 로깅은 추후 구현한다.
