# 2026-05-14 Wave 29 Hold Checkpoint — Smartphone deterministic 보류

## 결정

- `smartphone` 카테고리는 **deterministic narrow lane만으로는 현재 unlock 불가**로 본다.
- 이 상태는 **중요 보류**로 취급한다. 이후 제품 본류에서 smartphone broad 확대나 silent carrier 추정으로 우회하지 않는다.
- smartphone의 다음 정식 unblock 경로는 **AI L2 phase 1** 뿐이다.

## 근거

- Wave 28에서 `iPhone 15 Pro 128GB 자급제` narrow lane을 가장 dense한 sample로 검증했지만 validated row가 `0`이었다.
- broad `iphone-15-pro` 활성 row에서도 자급제 token이 `0건`이라, 한국 중고 iPhone 시장은 silent carrier state가 기본 패턴으로 확인됐다.
- explicit self-unlocked token 요구는 precision은 지키지만 recall이 `0`에 수렴한다.

## 보류 범위

- 보류 대상:
  - broad smartphone widening
  - silent carrier state를 deterministic으로 추정하는 parser/catalog patch
  - 자급제/언락 token 없는 iPhone을 narrow lane으로 구조 구제하는 시도
- 허용 대상:
  - AI L2 phase 1 설계/구현
  - smartphone 외 카테고리의 다양화/운영 검증

## 다음 우선순위

- smartphone은 **AI L2 phase 1 시작 전까지 보류 유지**
- 다음 제품 wave는 Apple 추가 다양화보다 **smartphone AI L2 phase 1**을 우선 검토한다.

## 추가 결정 — 마이닝은 종료가 아니라 선택적 보강 단계

- 현재 제품 본류는 `파서만 남은 상태`가 아니다. **선택적 마이닝 보강이 아직 필요하다.**
- 다만 초기처럼 카테고리 전체를 넓게 다시 캐는 단계는 지났다. 앞으로의 마이닝은 아래 3종만 허용한다.
  - `query_precision_repair`: 검색어 오염 제거가 우선인 lane
  - `structured_more_mining`: sample density가 부족한 exact lane 보강
  - `exact_lane_backfill`: 이미 맞는 SKU인데 활성 표본만 얇은 lane 보강

## 마이닝이 아직 필요한 근거

- `airpods_broad` / `earphone_discovered_broad` 검증에서, 마이닝은 단순 수집이 아니라 **SKU 분포, accessory/parts/bundle/proshop 노이즈 분리**의 핵심 단계로 확인됐다.
- `AirPods Pro 2 왼쪽/오른쪽` 같은 parts noise도 파서가 먼저 안 것이 아니라, **마이닝/재검증 과정에서 드러난 뒤 분류 규칙에 반영**됐다.
- `lg_gram_17_2024`는 parser patch보다 **query contamination 정리(마이닝 query repair)** 가 우선이라고 이미 확정됐다.
- `camera`, 일부 non-Apple monitor/speaker/home_appliance, `smartphone` broad family는 지금도 **sample density 또는 query precision 문제**가 남아 있어 마이닝 보강이 완전히 끝난 상태가 아니다.

## 운영 원칙

- Apple closed-set 성공 lane은 파서/카탈로그 패턴이 우세하므로 마이닝 비중이 작다.
- 반대로 `camera / smartphone / non-Apple 다양화`는 여전히 **마이닝 결과를 근거로 파서/카탈로그를 조정**해야 한다.
- 즉 앞으로는 `마이닝 -> 증거 확보 -> 파서/카탈로그 수동 강화 -> 재검증` 순서를 유지한다.
