# 2026-06-05 Wave 1144 — 당근 기본 피드 예산 필터 빈 결과 자동 복구

## 결정
- 기본 피드를 당근/가까운순으로 바꾼 뒤, 이전 세션에 저장된 예산 필터가 같이 적용되어 빈 피드가 되는 경우를 자동 복구한다.
- 현재 세션에서 사용자가 직접 예산을 선택한 경우에는 자동으로 풀지 않는다.

## 구현
- `src/components/explore-client.tsx`
  - 세션 내 예산 직접 선택 여부를 `budgetTouchedThisSessionRef`로 추적한다.
  - 기본 당근 요청에서 `budgetFilter !== all`이고 결과가 0개이며 사용자가 이번 세션에 예산을 직접 누르지 않았다면, 예산을 `all`로 한 번만 자동 해제한다.

## 원인
- 피드 기본값을 `source=daangn`, `sort=distance`로 바꾸면서 localStorage에 남아 있던 `15만원 이하` 예산 필터가 같이 적용됐다.
- 그래서 사용자는 “당근 먼저 보기”를 기대했는데 실제 요청은 `당근 + 가까운순 + 15만원 이하`가 되어 후보가 0개로 떨어질 수 있었다.

## 보류
- 당근 예산별 서버 후보 수를 미리 보여주는 별도 카운트 API는 추가하지 않았다.
