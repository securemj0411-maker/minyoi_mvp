# 2026-05-18 Wave 224 — 차익 소멸 추천은 사용자 화면에서 판매완료 처리

## 문제
- `/me`에서 현재 시세 재계산 결과 차익이 0원 이하로 내려가면 `추천 무효`로 표시했다.
- 이 문구는 사용자가 보기엔 "우리가 틀린 추천을 줬다"는 인상을 주며 신뢰를 깎는다.
- 다만 실제 번개장터 매물이 팔린 것은 아닐 수 있으므로 DB `listing_state`를 `sold_confirmed`로 위조하면 시세/라이프사이클 데이터 품질이 망가진다.

## 결정
- `marketStale=true`인 reveal은 사용자 `/me` 화면에서 판매완료 tombstone으로 접는다.
- DB의 실제 매물 상태는 바꾸지 않는다. `current_profit_*`와 `market_invalidated_at`은 계속 sync해서 내부적으로 차익 소멸 상태를 추적한다.
- 이후 표본이 쌓여 현재 차익이 다시 양수로 돌아오면 다음 `/me` 갱신에서 정상 카드로 복귀할 수 있다.

## 보류
- `market_invalidated_at`이 찍힌 reveal을 일정 시간 뒤 자동 숨김/아카이브할지 여부는 실제 베타 행동 데이터를 보고 결정한다.
