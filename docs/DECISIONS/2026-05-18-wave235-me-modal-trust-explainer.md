# 2026-05-18 Wave 235 — /me 상품 보기 신뢰 설명 UX

## 배경

사용자 요청: `/me` 상품 보기 모달이 토스처럼 누구나 이해하기 쉽게 "왜 이걸 추천했는지"와 "시세 그래프가 어떤 기준인지"를 설명해야 한다. 특히 시세가 비슷한 상태의 매물을 기준으로 잡힌다는 신뢰 표현이 필요하다.

## 결정

- 상품 보기 모달에 `왜 이걸 추천했나요?` 접이식 패널을 추가한다.
- 추천 이유는 내부 점수 대신 일반 사용자 언어로 나눈다.
  - 같은 모델로 묶었는지
  - 같은 상태 또는 미개봉/새상품 기준 시세인지
  - 판매수수료, 재배송비, 안전버퍼를 뺀 현재 차익인지
  - 판매완료/삭제 매물은 다시 검증하는지
- 시세 그래프 상단에 기준 설명을 추가해 "상태가 다른 매물을 섞어 시세를 부풀리지 않는다"는 신뢰 문구를 노출한다.
- 사용자 화면에 남아 있던 `band N` 내부 구간 표기는 제거한다.

## 반영

- `PackRevealModal`에 `RecommendationReasonPanel`과 `MarketGraphTrustLine`을 추가했다.
- 번개 시세 요약은 `번개 중고 매물 기준` 같은 뭉뚱그린 문구 대신 `번개 {상태} 매물 기준`으로 바꿨다.
- 다나와 reference 상품은 다나와 새 가격 기준과 번개 미개봉 추이를 구분해 설명한다.
- `/me` 모달 계약 테스트에 신뢰 설명 문구와 내부 band 표기 제거 검증을 추가했다.

## 보류

- 이번 작업은 UX 설명과 기존 데이터 표현을 정리한 것이다. 시세 산식, sample 집계, lifecycle write-through 정책은 변경하지 않는다.
- 실제 Toss급 마이크로 인터랙션이나 애니메이션은 다음 UI polish wave에서 별도 검토한다.
