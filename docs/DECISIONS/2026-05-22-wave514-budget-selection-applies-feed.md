# Wave 514 — 첫 예산 선택 즉시 피드 반영

## 문제
- 첫 가입 온보딩에서 예산을 선택해도 피드가 예산대에 맞게 바뀌지 않았다.
- 클라이언트는 `budgetFilter`를 저장했지만 `/api/packs/pool` 요청에는 `budget` 파라미터를 보내지 않았다.
- 또한 클라이언트 표시 단계에서 예산 필터 결과가 6개 미만이면 전체 후보를 다시 보여주는 fallback이 있어, 사용자가 선택한 예산이 무시된 것처럼 보였다.

## 결정
- 예산 선택은 서버 pool 요청에도 반영한다.
- 선택한 예산 필터는 클라이언트 표시에서도 그대로 적용한다. 부족하면 부족한 상태를 보여주고, 사용자가 필터를 초기화하거나 더 찾아보게 한다.

## 구현
- `budgetApiParam()`으로 `150000/300000/500000`을 서버 API의 `150k/300k/500k`로 매핑했다.
- `loadPool()`이 `budgetFilter` 변경에 반응하고 `/api/packs/pool?budget=...`을 호출하게 했다.
- 예산 필터 결과가 적을 때 전체 후보로 되돌리는 `MIN_BUDGET_FILTER_RESULTS` fallback을 제거했다.

## 보류
- 예산 필터 결과가 0개일 때 별도 UX 문구를 더 친절하게 바꾸는 작업은 후속으로 남긴다.
