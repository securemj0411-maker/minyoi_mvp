# Wave 1157 - 예산 필터 첫 로드 후보 스캔 폭 보정

## 결정
- `budget` 파라미터가 있는 피드 요청은 첫 로드여도 refresh와 같은 `REFRESH_READY_CANDIDATE_LIMIT` 폭으로 후보를 다양화한다.

## 이유
- 기존 구조는 예산 필터가 있어도 raw 후보 fetch는 넓게 했지만, 첫 로드 최종 후보 다양화는 25개만 통과시켰다.
- 15만원 이하처럼 좁은 조건에서는 앞 25개가 build-time 수익 재계산, lifecycle, dedupe에서 탈락해 빈 화면이 뜨고, 새로고침은 500개를 훑기 때문에 뒤늦게 정상 후보가 나왔다.
- 같은 조건에서 새로고침해야만 후보가 나오는 것은 사용자에게 “매물 없음”으로 잘못 보이는 심각한 UX 버그다.

## 보류
- 후보 스캔 폭을 예산별로 더 세밀하게 조정하는 성능 튜닝은 실제 응답 시간 관찰 후 진행한다.
