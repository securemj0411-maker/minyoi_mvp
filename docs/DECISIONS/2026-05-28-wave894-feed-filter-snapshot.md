# Wave 894 — 피드 필터는 현재 snapshot 기준으로 적용

## 문제

`/me` 피드에서 출처/정렬 셀렉트를 바꾸면 현재 화면의 매물을 필터링하는 것이 아니라 `/api/packs/pool`을 다시 호출했다.

예: 현재 피드에 번개장터 매물이 보이는 상태에서 `최신순`을 누르면 새 서버 풀을 받아 당근 매물이 갑자기 섞여 나옴.

사용자 인식은 "지금 보고 있는 피드 안에서 정렬/필터" 이므로 화면이 순간이동하는 느낌이 난다.

## 결정

- 예산 필터만 서버 재조회 대상으로 유지한다.
  - 예산은 후보군 자체가 달라지는 필터라 서버 재조회가 자연스럽다.
- 출처, 카테고리, 정렬은 현재 로드된 `items` snapshot 안에서 client-side 로 적용한다.
- `더 찾아보기`는 현재 출처 필터가 켜져 있을 때만 `source`를 서버에 넘긴다.
  - 사용자가 "당근만"을 보고 있는 상태에서 더 찾으면 당근 후보를 우선 추가로 받는다.
  - `최신순` 같은 정렬은 추가 로드 요청에 넘기지 않는다.
- 필터 결과가 비었을 때 문구를 "현재 피드에 조건 매물이 없음"으로 바꿔, 서버 풀 부족처럼 오해하지 않게 한다.

## 보류

- 카테고리별 추가 로드를 서버에 넘기는 것은 보류.
  - 지금 API가 source/budget 중심으로 안전하게 동작하고, 카테고리는 기존에도 client-side 정책이었다.
- 전체 피드 snapshot 크기 확대(30 → 60 등)는 별도 UX/성능 검토 후 결정.
