# 2026-05-20 Wave 416 — 크레딧 충전 페이지 단순화

## 결정
- `/plans`는 구독/요금제 설명 페이지가 아니라 크레딧 충전 페이지로 운영한다.
- 사용자에게 보이는 상품은 `20개 3,900원`, `200개 19,900원`, `500개 39,900원`의 단건 충전권으로 표현한다.
- 모바일 첫 화면에서 충전권 선택과 `충전하기` 버튼이 바로 보이도록 비교표, 현재 상품 상태, 긴 FAQ를 제거한다.
- 무료 상세보기 한도 도달 CTA의 `Plus`/`PRO` 표현은 크레딧 충전 표현으로 대체한다.

## 변경
- `src/app/plans/page.tsx`를 충전 카드 3개 중심의 모바일 우선 화면으로 재구성했다.
- `/plans`에서 checkout으로 넘어갈 때 `plan=plus/pro` 대신 `credits=200/500` 형태의 URL을 사용한다.
- 첫 수정안의 헤더/카드 타이포가 과하게 커서, 모바일 기준 h1 20px / 충전 수량 24px / 카드 높이 약 134px의 리스트형 UI로 다시 낮췄다.
- 하단 빈 공백이 커 보여서 환불 안내 아래에 `충전 전 확인` 기본 안내 섹션을 추가했다.
- 충전권별로 “관심 매물 확인”, “후보 추리기”, “대량 탐색” 용도를 보여주고, 하단에 상세페이지에서 제공되는 예상 순익/시세 그래프/비교 매물/원본 링크 안내를 추가했다.
- 충전권 표기는 `20개`가 아니라 `20크레딧`처럼 명확히 쓰고, 단가는 `195원/1크레딧` 형태로 표시한다.
- 충전 수량은 `[코인 아이콘] 20 크레딧`처럼 크레딧 로고를 숫자 왼쪽에 배치해 더 빠르게 인지되도록 한다.
- 결제 금액은 CTA보다 과하게 튀지 않도록 `font-black`에서 `font-bold`로 낮춘다.
- 수익 보장, 평균 수익, 본전 보장처럼 표시광고 리스크가 큰 표현은 쓰지 않고 실제 거래 결과는 조건에 따라 달라진다는 문구를 둔다.
- `src/components/explore-client.tsx`의 `Plus로 계속 보기`, `PRO` chip, 유료 패키지 문구를 크레딧 충전 문구로 바꿨다.
- `src/app/billing/checkout/checkout-client.tsx`의 결제 요약에서 하루 한도 대신 `상세보기 1회 = 1크레딧` 사용 기준을 보여준다.
- `src/lib/plan-config.ts`의 충전권 feature 문구에서 구독형/하루 한도 느낌을 줄였다.

## 보류
- 내부 plan key(`starter`, `plus`, `pro`)와 RPC 인자는 기존 결제/권한 흐름 호환을 위해 그대로 둔다.
- 크레딧 만료일, 부분 환불 계산 UI, 영수증/거래명세 UI는 PG 심사 기본 페이지 이후 별도 작업으로 둔다.
