# 2026-05-22 Wave 530 — 예산 필터 피드 소진 UX와 무한 피드 속도

## 결정
- 예산 필터가 켜진 상태에서는 서버가 더 높은 가격대로 조용히 fallback하지 않는다.
- 예: 15만원 이하 피드에서 더 이상 후보가 없으면 30만원/50만원 매물을 몰래 가져오지 않고, "15만원 이하 조건은 오늘 여기까지"라고 안내한다.
- 사용자는 CTA로 가격대를 한 단계 넓히거나 전체 가격대로 전환할 수 있다.
- 크레딧 보유자 무한 피드 sentinel은 더 이른 시점에 다음 후보를 요청해, 하단에서 로딩 문구를 오래 보는 느낌을 줄인다.

## 이유
- 기존 구조는 15만원 이하 후보가 없을 때 서버가 30만원/50만원/전체 후보까지 재조회하고, 프론트에서 다시 15만원 필터로 숨길 수 있었다.
- 이 경우 사용자는 새 매물이 안 붙는 것처럼 보는데 서버는 비싼 후보를 가져오느라 느려진다.
- 예산 필터는 사용자의 실제 구매 가능 금액과 직결되므로, 조용히 넓히기보다 사용자에게 다시 선택권을 주는 편이 신뢰에 맞다.

## 보류
- 무한 피드 자체의 서버 비용 절감은 별도 작업이다. 후보 페이지네이션용 cursor/cache, budget별 precomputed slice, light response path를 검토한다.
- "오늘 이 가격대 후보 수"를 숫자로 보여주는 기능은 pool count 쿼리 비용을 보고 결정한다.
