# 2026-05-19 Wave 278 — /me 모바일 상세를 페이지형 전체화면으로 전환

## 문제
- /me 상품 상세는 모바일에서 거의 전체 화면 모달처럼 보이지만, 실제 URL/뒤로가기 동작은 모달이라 사용자가 페이지 이동으로 착각할 수 있다.
- 모달 높이 제한(`88vh`)과 외곽 여백 때문에 작은 화면에서는 핵심 정보와 그래프가 더 좁게 보였다.

## 결정
- 모바일에서는 상세 모달을 `h-dvh` 전체 화면, `rounded-none`, `border-0`, `p-0` 구조로 바꿔 페이지처럼 보이게 했다.
- PC에서는 기존 `sm:max-h-[88vh]`, `sm:max-w-6xl`, rounded modal UI를 유지한다.
- 모바일 카드 클릭 시 `?reveal=<pid>&preview=<mode>` URL state를 push한다.
- 브라우저 뒤로가기는 상세를 닫고 /me 목록으로 돌아오게 했다.
- 헤더 버튼 문구도 모바일에서는 `뒤로`, PC에서는 `닫기`로 다르게 보여준다.

## 보류
- `/me/reveals/[pid]` 실제 라우트 분리는 보류했다. 지금은 상세 UI가 `PackRevealModal` 내부에 크고, 페이지/모달 공용 컴포넌트 분리까지 하면 변경 범위가 커진다.
- 이번 wave는 사용자 혼란과 화면 밀도 문제를 줄이는 page-like shell + URL/back 동작에 집중했다.
