# 2026-06-06 Wave 1176 — 모바일 피드 점진 로딩

## 결정
- `/api/packs/pool`에 `limit` 파라미터를 추가해 첫 응답 개수를 조절할 수 있게 했다.
- 모바일 피드 첫 로딩은 기본 6개만 먼저 받아 즉시 렌더하고, 이후 24개를 조용히 백그라운드로 append한다.
- 백그라운드 보강 요청은 실패해도 이미 보이는 피드를 에러 화면으로 덮지 않는다.

## 이유
- 모바일에서는 첫 화면에 보이는 카드가 3~5개라 30개 전체 응답을 기다리는 체감이 나쁘다.
- 한 개씩 API를 쪼개면 요청 수와 정렬 흔들림이 커지므로, 작은 첫 chunk + 큰 보강 chunk가 안정적이다.
- 당근 근처 피드는 누락 방지가 중요해서 첫 chunk도 최소 25개 후보를 후처리하고, 보강 요청에서는 기존 깊은 스캔을 유지한다.

## 보류
- 실제 HTTP streaming/SSE 방식은 이번에 도입하지 않았다. 필요하면 이후 feed RPC 또는 edge streaming으로 별도 설계한다.
