# 2026-05-21 Credit holder infinite feed

## Context
- 30개 추천 풀은 서버 배치 단위로는 유지하되, 크레딧 보유 사용자가 매번 `더 찾아보기`를 눌러야 하는 UX는 피곤하다.
- 과금 지점은 탐색이 아니라 상세 분석 열람이어야 한다.

## Decision
- 크레딧을 1개 이상 보유한 사용자는 피드 탐색 쿨다운을 우회한다.
- 클라이언트는 크레딧 피드 모드일 때 바닥 sentinel을 IntersectionObserver로 감지해 다음 30개를 자동 append한다.
- 자동 append는 새 매물 첫 카드로 강제 스크롤하지 않는다. 사용자가 당근 피드처럼 계속 내려가며 보도록 둔다.
- 크레딧은 계속 기존 정책대로 새 상세 분석을 열 때만 차감한다.
- 무료 사용자가 쿨다운 중 `더 찾아보기`를 누르면, "크레딧 1개 이상이면 피드 탐색 무제한" 혜택을 먼저 설명한다.
- 이 혜택 안내에서는 피드 탐색은 차감 0개이고, 차감 지점은 새 상세 분석 열람이라는 점을 명확히 노출한다.
- 이전 "지금 즉시 매물 N건은 크레딧 충전 사용자 전용" 문구와 6시간 신선 매물 잠금 통계는 현재 정책과 맞지 않아 제거한다.

## Deferred
- 계정 등급별 피드 품질 차등, 더 큰 page size, 서버 cursor 방식은 후속 작업으로 둔다.
- 크레딧 0개가 되는 순간 이미 로드된 피드를 회수하지는 않는다. 이후 자동 append만 중단한다.
