# Wave 905 — 첫 피드 온보딩 3단계 축소

## 문제
- 가입 직후 흐름은 동네 설정 온보딩 이후 `/me` 첫 피드 온보딩으로 이어진다.
- `/me` 첫 피드 온보딩이 4단계였다.
  - 의심 mirror
  - 같은 상태 비교 예시
  - 풀 통계 / 필터링 숫자
  - 예산 설정
- 풀 통계 단계는 "전체 중고 상품 N건 중 어려운 상품을 걸렀다"는 내부 성과 설명이라 사용자의 다음 행동과 직접 연결이 약했다.
- 이 단계 때문에 `/api/public/safety-stats` fetch와 "확인 중" 로딩도 첫 가입 경험에 섞였다.

## 결정
- `/me` 첫 피드 온보딩은 4단계에서 3단계로 줄인다.
- 제거 대상은 풀 통계 / 필터링 숫자 단계다.
- 남기는 단계는 `의심 → 같은 상태 비교 예시 → 예산 설정`이다.
- 동네 설정 온보딩(`/onboarding/home-region`)은 그대로 유지한다.

## 구현
- `src/components/explore-client.tsx`
  - `FirstFeedOnboardingCard` step total을 4에서 3으로 변경.
  - `step === 2`의 풀 통계 화면 제거.
  - 예산 설정을 마지막 3번째 단계로 이동.
  - 첫 온보딩에서 `/api/public/safety-stats` 호출 제거.
  - 사용하지 않는 safety stats 타입/상태/헬퍼 제거.

## 보류
- 풀 통계 메시지를 완전히 폐기하지는 않는다. 추후 필요하면 피드 상단의 작고 닫을 수 있는 신뢰 배지로 재배치할 수 있다.
- 예산 설정 문구 자체를 더 짧게 만드는 작업은 별도 UX 패스로 남긴다.
