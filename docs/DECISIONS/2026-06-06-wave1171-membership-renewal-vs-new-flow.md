# Wave 1171 — 신규 가입과 연장 결제 흐름 분리

## 배경
- 멤버십 신규 가입과 연장 결제 흐름이 같은 컴포넌트를 쓰면서 의미가 섞였다.
- 신규 가입은 지역 티오를 먼저 확보하는 흐름이라 7분 입금 예약 압박이 필요하다.
- 연장은 이미 활성 멤버가 기존 만료일 뒤에 기간을 붙이는 흐름이라 7분 안에 입금하지 않으면 예약 취소된다는 표현/상태가 맞지 않는다.

## 결정
- 신규 신청만 7분 미입금 자동 만료 대상으로 유지한다.
- 연장 예약은 7분 자동 만료 대상에서 제외하고, 입금 전까지 기간 변경/연장 취소가 가능한 pending 예약으로 둔다.
- 프론트 결제 모달은 신규와 연장 문구를 분리한다.
  - 신규: 자리 확보, 멤버십 승인, 5분 내 가입 반영.
  - 연장: 연장 입금, 연장 승인, 기존 만료일 뒤에 기간 추가.
- 관리자 페이지도 "입금 전 예약 전체가 7분 만료"처럼 보이지 않게 신규/연장 의미를 구분한다.

## 보류/주의
- 실제 계좌 입금 감지는 아직 수동 `입금했어요` + 5분 자동승인 흐름이다.
- 연장 pending을 장기간 유지할지, 특정 기간 이후 운영자 정리 대상으로 볼지는 추후 운영 데이터 보고 결정한다.
