# 2026-05-22 Wave 534 — 모바일 네비 크레딧 우선순위

## 결정
- 로그인 모바일 상단 네비에서 `대시보드` 버튼은 제거한다.
- 모바일 상단 오른쪽에는 보유 크레딧 칩만 상시 노출한다.
- 모바일 중앙 `득템잡이` 링크는 로그인 상태에서 `/me`, 비로그인 상태에서 `/`로 보낸다.
- 햄버거 메뉴는 로그인 사용자 기준 `추천 피드 → 크레딧 충전 → 서비스 안내` 순서로 재정렬한다.

## 구현
- 상단 모바일 우측 액션을 크레딧 칩 하나로 단순화했다.
- 모바일 햄버거 헤더의 브랜드 반복(`M`, `득템잡이`)을 제거하고, 기존 하단 프로필 칩을 상단 프로필 버튼으로 이동했다.
- 햄버거 메뉴 상단에 보유 크레딧 강조 카드를 추가하고, 계정 관리는 하단으로 내렸다.
- 모바일 메뉴 링크에는 짧은 보조 설명을 붙여 각 항목의 역할을 바로 알 수 있게 했다.

## 보류
- 크레딧 칩 클릭 시 `/plans` 직행 대신 바텀시트를 여는 방식은 추후 사용 내역/충전 패키지 UX를 같이 정리할 때 검토한다.
