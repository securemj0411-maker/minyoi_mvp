# Wave 510 — 상세페이지 시세 그래프 MVP 숨김

## 문제
- 상세페이지의 `시세 그래프 · 시장 분석`은 카테고리/SKU별 데이터 준비도가 충분히 맞아야 신뢰를 줄 수 있다.
- MVP 현재 상태에서 모든 SKU에 같은 깊이의 그래프/회전/유입 해석을 노출하면, 비교 매물보다 오히려 설명 책임이 커지고 신뢰 리스크가 생긴다.

## 결정
- 상세 숫자 모드에서는 `시세 그래프 · 시장 분석` 블록을 일단 숨긴다.
- `시세 비교 매물`은 유지한다. 비교 매물은 이미 API outlier 제거 후 가격 높은 순으로 정렬한다.

## 구현
- `RevealCardItem` 상세 숫자 모드에서 `MarketHistoryChart`, `MarketGraphTrustLine`, `SkuListingFlowMini` 묶음 렌더링을 제거했다.
- `/me` 계약 테스트는 비교 매물 섹션이 남아 있고 상세 그래프 마커가 없는지를 확인하도록 조정했다.

## 보류
- SKU/카테고리별 그래프 준비도 기준, 표본 수 기준, source별 거래가/호가 분리 기준을 정한 뒤 그래프 재노출 여부를 다시 결정한다.
