# 2026-05-17 Daily Market Brief — 보류 (텔레그램만 가능, 사용자 카톡 결정 전)

## 컨텍스트

Master plan Phase 0 항목 — "매일 아침 시장 브리핑 봇" (habit ritual 형성, retention factor).

기획:
- 오전 8시 한 줄 push: "에어팟 -2.1%, 갤워치 +0.8%, D-12 아이폰 17"
- cron + market_history_daily 집계로 1~2일 구현

## 보류 이유

사용자 응답: **"매일 아침 시장알림은 보류 나 지금 텔레그램 밖에 안됌"**

- 인프라 가용: 텔레그램 봇만
- 한국 일반인 사용자는 텔레그램 사용률 매우 낮음 — 카톡이 메인
- 카톡 알림톡 = 비즈 채널 등록 + 발신 프로필 + 알림톡 템플릿 승인 절차 필요 (1~2주 + 비용)
- 채널 결정 (카톡 비즈 채널 / 카톡 친구톡 / 웹 push / 이메일) 없이 텔레그램만 박으면 **사용자 도달율 매우 낮음**

## 결정

**채널 결정될 때까지 보류**. 텔레그램만 박으면 효용 미미.

## 재개 조건

다음 중 하나:
1. 카톡 알림톡 비즈 채널 등록 완료
2. 카톡 친구톡 가능한 채널 확보
3. 웹 push (FCM 등) 인프라 박힘
4. 이메일 newsletter 결정 (간단하지만 retention factor 낮음)

## 다음에 까먹지 않기 위한 trigger

- 사용자 base > 100명 도달 시점에 카톡 비즈 채널 등록 검토
- 결제 인프라 박을 때 같이 알림 인프라도 검토 (현재 토스/카카오페이 검토 중이면 그 시점 같이)
- 신규 가입 funnel 측정 시 "재방문" drop이 큰지 확인 — 크다면 알림 채널 우선순위 올림

## Trade-off 명시

- 보류 = habit ritual retention factor 1순위 늦어짐
- 단 사용자 도달율 없는 채널은 "박았다" 만 있고 실제 효과 0 — 더 큰 burn
