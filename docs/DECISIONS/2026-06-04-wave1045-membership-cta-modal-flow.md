# 2026-06-04 Wave 1045 — membership CTA and modal application flow

## Trigger

`/plans`에서 CTA가 너무 아래에 있고, 플랜 카드가 페이지 본문에 바로 노출되어 신청 흐름이 산만했다.

또 업셀 모달에서 특별가 카드를 누르는 순간 바로 신청 API가 호출되어, "선택"과 "최종 신청"이 분리되지 않았다.

## Decisions

- 가격/CTA는 히어로 카피 바로 아래에 둔다.
- `/plans` 본문에는 1/3/6/12개월 카드 전체를 직접 노출하지 않는다.
- CTA를 누르면 플랜 선택 모달을 띄운다.
- 모달에서 기간을 고른 뒤 별도 `신청하기` 버튼을 눌러야 다음 단계로 간다.
- 업셀 모달의 특별가 카드는 선택만 한다.
- 업셀 최종 신청은 하단 버튼을 눌렀을 때만 API를 호출한다.

## Applied

- `/plans` 상단에 `월 33,000원꼴` note와 신청 CTA를 올렸다.
- `MembershipApplicationClient`에 플랜 선택 모달을 추가했다.
- 업셀 모달을 `카드 선택 → 최종 신청 버튼` 구조로 바꿨다.
- pending 신청 상태는 기존처럼 DB row를 서버에서 읽어 표시한다.

## Deferred

- 페이지에 추가 시각 자산/SVG를 넣는 디자인 강화는 다음 wave로 보류했다.
- 업셀 만료의 서버 측 검증은 결제 자동화 단계에서 함께 처리한다.
