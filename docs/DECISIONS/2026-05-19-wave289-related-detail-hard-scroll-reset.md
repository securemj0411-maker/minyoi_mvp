# 2026-05-19 Wave 289 — 관련 매물 전환 스크롤 리셋 강화

## 결정
- 관련 매물 클릭 순간에 상세 스크롤을 즉시 `top=0`으로 내린다.
- 새 상세 PID/preview seed가 반영된 뒤 `useLayoutEffect`와 `requestAnimationFrame`으로 한 번 더 리셋한다.
- 내부 스크롤 컨테이너에 `key={activeRevealPid}`를 걸어 PID 변경 시 DOM 스크롤 상태가 유지되지 않게 했다.

## 이유
- 기존 Wave 288은 PID 변경 이후 effect에만 의존해 브라우저 스크롤 앵커링/렌더 타이밍에 따라 사용자가 변화를 못 느낄 수 있었다.
- 관련 매물 전환은 페이지 이동에 가까우므로, 이전 스크롤 위치를 보존하면 안 된다.

## 보류
- 전환 토스트나 상단 하이라이트는 아직 넣지 않았다.
- 실제 개별 상세 라우트 분리는 계속 보류한다.
