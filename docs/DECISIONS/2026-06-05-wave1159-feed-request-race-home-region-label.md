# 2026-06-05 Wave 1159 — 피드 요청 레이스와 기준 동네 노출

## 결정
- 추천 피드의 `/api/packs/pool` 요청에 sequence guard를 추가했다.
- 예산/출처/정렬 필터 변경 중 이전 요청이 늦게 도착해 최신 필터 결과를 덮어쓰지 못하게 했다.
- 피드 필터 요약과 필터 시트에 현재 저장된 기준 동네를 표시한다.

## 배경
- cau pool에는 동작구/서초구 15만원 이하 당근 ready 후보가 실제로 존재했다.
- 사용자 피드에서는 `15만원↓ + 당근 + 가까운 순` 상태에서 빈 상태가 뜨고, 새로고침 후에는 후보가 보이는 현상이 있었다.
- 원인은 DB 표본 부족보다 클라이언트 요청 레이스 가능성이 높았다. 이전 필터 요청 응답이 최신 상태를 덮으면, 클라이언트 필터에서 현재 예산과 맞지 않는 snapshot이 걸러져 빈 화면이 될 수 있다.
- 또한 화면에 기준 동네가 보이지 않아 실제 home region이 `동작구`인지 `서초구`인지 사용자가 확인할 수 없었다.

## 보류
- cau pool에서 raw listing이 `disappeared`인데 pool row가 ready로 남는 항목은 별도 운영 정리 대상이다.
