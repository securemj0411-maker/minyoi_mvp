# 2026-05-18 Wave 226 — 더 찾아보기 결과를 보관함에 직접 반영

## 배경
- `/me`의 "더 찾아보기" 모달에서 카드를 뽑은 뒤 별도 결과 모달이 뜨면, 사용자는 이 카드가 보관함에 저장된 것인지 스크린샷을 해야 하는지 헷갈릴 수 있다.
- 실제 UX 목표는 "새 추천을 뽑았다"가 아니라 "내 추천 보관함에 새 후보가 추가됐다"는 사실을 즉시 확인시키는 것이다.

## 결정
- `/me` 안에서 호출되는 `RecommendationWorkspace`는 성공 결과 모달을 띄우지 않는다.
- 성공 시 `PACK_REVEALS_UPDATED_EVENT`로 보관함 목록을 즉시 갱신하고, "더 찾아보기" 모달을 닫는다.
- 새로 추가된 카드는 보관함 맨 위에 낙관적으로 삽입하고 12초 동안 rose ring/glow 및 "방금 추가" 배지를 보여준다.
- 필터/정렬 때문에 새 카드가 숨어 보이지 않지 않도록 pack open 이벤트를 받으면 검색어를 비우고 최신순으로 돌린다.

## 보류
- 실패/환불/재고 부족 케이스는 아직 별도 상세 모달을 쓰지 않고, `RecommendationWorkspace` 내부에 짧은 인라인 안내만 표시한다.
- pack open 직후 서버 재조회 타이밍은 다음 개선에서 `loadItems`에 explicit query/sort/page override를 넣어 더 정교하게 맞출 수 있다. 현재는 이벤트 payload 기반 낙관적 삽입으로 사용자가 즉시 확인할 수 있게 했다.
