# 2026-06-04 Wave 1049 — membership deposit confirmation flow

## Trigger

`/plans`에 `1. 신청 → 2. 지역 조회 → 3. 입금 후 활성` 절차 카드가 있었지만, 로그인 후 plans 페이지에 들어온 사용자에게는 중복 설명처럼 보이고 실제 궁금한 입금 이후 행동과 대기 시간이 드러나지 않는다는 피드백이 있었다.

## Decisions

- 추상적인 신청 절차 카드는 제거한다.
- 사용자가 궁금해하는 `송금 후 무엇을 누르는지`, `얼마나 기다리는지`, `승인 전 변경이 가능한지`를 직접 보여준다.
- 입금 완료 버튼은 자동 승인 버튼이 아니라 운영자에게 입금 확인 요청을 보내는 버튼으로 둔다.
- 권한 활성화는 계속 cau 운영자 페이지의 승인 버튼으로만 처리한다.
- 카피상 기대 시간은 `보통 3분 내 확인`으로 안내하지만, 자동 멤버십 승인 cron은 이번 작업에서 만들지 않는다.

## Applied

- `/plans`의 `신청/지역 조회/입금 후 활성` 카드들을 `입금했어요 버튼/보통 3분 내 확인/승인 전 변경 가능` 카드로 교체했다.
- pending 예약 카드에 `입금했어요` 버튼을 추가했다.
- `POST /api/membership/deposit-notify`를 추가해 사용자 입금 확인 요청을 운영자 텔레그램으로 보낸다.
- 입금 확인 요청 시 `mvp_membership_applications.admin_note`에 `user_deposit_confirmed`와 텔레그램 알림 결과를 남긴다.

## Deferred

- 입금 후 3분 자동 승인.
- 멤버십 신청 전용 텔레그램 inline approve/reject 버튼.
- 승인 지연 시 사용자에게 별도 카톡/문자 상태 알림.
