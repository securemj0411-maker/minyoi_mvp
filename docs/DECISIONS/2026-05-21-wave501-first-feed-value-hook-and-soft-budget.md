# 2026-05-21 Wave 501 — 첫 피드 가치 훅 + 소프트 예산 필터

## 결정
- 상세보기 쉬운모드의 첫 장에 반복되던 "오늘 걸러낸 매물" 안내는 첫 방문 피드 온보딩으로 옮긴다.
- `/api/packs/pool`에는 예산/성향을 다시 보내지 않는다. 희귀 pool 손실을 막기 위해 서버는 넓게 가져오고, 예산은 클라이언트에서만 우선 필터링한다.
- 첫 방문자는 오늘 추천 풀에서 걸러낸 매물 수와 예산 칩을 먼저 본다. 예산은 피드 상단에서 언제든 변경 가능하게 둔다.

## 구현
- `ExploreClient`에 첫 방문 가치 카드 추가:
  - `/api/public/safety-stats`로 전체 추천 풀 통계를 불러온다.
  - 돈 안 되는 것, 거래 주의 신호, 상품 확인 필요 항목을 간단히 보여준다.
  - `15만원 이하`, `30만원 이하`, `50만원 이하`, `상관없음` 예산 칩을 제공한다.
- 예산 선택은 `localStorage`에 저장하고 화면 표시에서만 적용한다.
- 상세 쉬운모드는 인트로를 제거하고 바로 비교 매물부터 시작한다.

## 보류
- 가입 플로우 자체의 서버 저장형 예산 프로필은 보류한다.
- 예산 안 매물이 없을 때는 빈 화면 대신 전체 후보를 보여준다. 추후에는 "예산 안 매물이 적어서 전체도 같이 보여드려요" 같은 안내를 붙일 수 있다.
- parser, comparable key, catalog 강화는 다른 세션에서 진행 중이라 이번 wave 범위에서 제외했다.
