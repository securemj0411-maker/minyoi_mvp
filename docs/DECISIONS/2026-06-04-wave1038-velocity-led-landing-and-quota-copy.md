# 2026-06-04 Wave 1038 — Velocity 중심 비회원 랜딩과 쿼터형 멤버십 설득

## 결정

- 비회원 메인 랜딩의 미리보기 샘플은 단순 판매완료/종료 샘플이 아니라 `판매 회전 속도`가 관측된 상품만 노출한다.
- 메인 CTA는 `로그인하고 보기`보다 `로그인하고 신청` 흐름으로 둔다. 로그인 후 미승인자는 `/plans`로 이어지게 한다.
- `/plans`는 “프리미엄이라 심사”보다 “돈 되는 매물이 희소하고, 당근은 지역 제약이 있어 쿼터를 관리한다”는 설명을 앞세운다.

## 구현

- `/api/preview-pool`에서 `mvp_market_velocity_daily`의 usable velocity가 있는 comparable key만 candidate로 남긴다.
- 랜딩 preview chip은 셀러 후기보다 `평균 N시간/일 내 판매`를 우선 노출한다.
- 비회원 랜딩의 CTA와 미리보기 카드 링크는 `/login?next=/plans`로 보낸다.
- `/plans`에 희소성, 지역성, 쿼터 관리 설명을 추가했다.
- landing/membership contract test를 추가해 velocity 없는 preview 회귀와 옛 무료/베타 문구 회귀를 막는다.

## 보류

- 실제 승인 운영 방식은 아직 텔레그램 신청 알림 기반이다. “24시간 내 카톡 응답 후 결제 링크” 같은 운영 플로우 자동화는 별도 작업으로 남긴다.
- 실제 300명 hard cap은 아직 적용하지 않았다. 현재는 포지셔닝/쿼터 관리 카피로만 운영한다.
