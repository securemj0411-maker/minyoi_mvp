# 2026-05-18 Wave 242 — /me 매물별 추천 이유 전면화

## 배경

사용자 지적: `/me` 상품 모달의 `왜 이걸 추천했나요?`가 모든 매물에 같은 원칙 설명처럼 보인다. 사용자는 "이 매물만의 특징"을 보고 싶어 한다.

## 결정

- `왜 이걸 추천했나요?`의 메인 영역은 공통 엔진 설명이 아니라 매물별 feature card로 바꾼다.
- feature card는 현재 차익, 시세 대비 할인율, 회전 속도, 오늘 유입량, verdict chip의 good 신호, 셀러 신호를 우선 사용한다.
- 기존의 "같은 모델로 묶음 / 비용 차감 / 판매완료 재검증" 같은 공통 원리는 `계산 기준 보기` 접힘 영역으로 이동한다.
- chip 기반 `좋은 점 / 확인할 점` 영역은 유지하되, 메인 추천 이유보다 보조 근거로 둔다.

## 보류

- LLM이 매물 설명을 문장으로 재요약하는 Advisor형 설명은 보류한다. 지금은 이미 있는 구조화 신호를 UI에 정확히 연결하는 것이 더 안전하다.
- 실제 상품별 copy A/B 테스트는 아직 하지 않는다. 먼저 boilerplate 제거와 매물별 feature visibility를 고정한다.

## 다음 작업

1. `/me`에서 실제 매물 몇 개를 열어 feature card가 비어 보이지 않는지 확인한다.
2. feedback에서 "왜 추천인지 이해됨/안 됨" 신호를 따로 받을지 검토한다.
